## Cron Job 진단 결과

영준이형님이 cron job이 제대로 안 돌아가는지 확인해달라고 하심.

### 실행된 Job들 (state에 lastRunAtMs 있음)
- `night-music-off` - lastStatus: "ok"
- `morning-music` - lastStatus: "ok"  
- `Moltbook 브리핑` - lastStatus: "ok"

### 실행 안 된 것 같은 Job들 (state: {})
- 머슴 활동 (오전/오후/저녁) - 여러 개 중복됨
- 아침 음악 (데이식스 재즈)
- GitHub 백업
- 노래 끄기 (one-time)

### 발견한 문제점
1. **중복 job들**: 머슴 활동 job이 여러 개 있음 - 일부는 `sessionTarget: "main"` + `payload.kind: "systemEvent"` (구버전), 일부는 `sessionTarget: "isolated"` + `payload.kind: "agentTurn"` (신버전)
2. **조합 규칙**: 
   - `sessionTarget: "main"` → `payload.kind: "systemEvent"` 필수
   - `sessionTarget: "isolated"` → `payload.kind: "agentTurn"` 필수
3. 구버전 형식의 job들이 제대로 실행 안 되고 있을 가능성
